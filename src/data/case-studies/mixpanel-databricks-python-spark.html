Lifting campaign ROI 21% by streaming 300K subscribers into GA4.

Subscription behavior lived outside GA4 and the 300K-user dataset exceeded what Excel could process, leaving marketing unable to segment campaigns by real subscriber behavior. I built a Python and Spark pipeline on Databricks to stream that data into GA4 continuously.

02 — The Obstacles

What stood between raw data and GA4

Getting subscription data into GA4 meant solving three problems at once: querying at scale, reshaping for GA4's event format, and delivering on a schedule.

QUERY

Large-scale querying needed Spark

Efficiently pulling subscription data from source databases at 300K-user scale required distributed query execution.

300K users to query = single-machine tools fall over
FORMAT

GA4 requires a specific event shape

Raw subscription records had to be transformed and batched into the event and user-property format GA4's API expects.

Raw subscription records = unusable by GA4's API as-is
DELIVERY

One-off exports wouldn't scale

Marketing needed continuous, automated delivery of fresh subscription data, not manual periodic exports.

Manual exports = GA4 data goes stale between runs
03 — The Solution

A Python + Spark pipeline streaming into GA4

Built on Databricks, the pipeline queries, batches, transforms, and ships subscription data to GA4 on an automated schedule.

1
Query at scale database → Spark SQL

Used Spark SQL on Databricks to efficiently query subscription data for 300K users directly from source databases.Spark SQL · Databricks

2
Batch and transform with Python

Broke the 300K-user dataset into manageable batches with Spark, then transformed records into GA4's required event format with Python.Python · Spark batching

3
Stream to GA4 via Measurement Protocol

Used the ga4mp API in Python to send transformed events and user properties into GA4 as granular, real-time records.ga4mp · Python

4
Automated, continuous scheduling

Scheduled the pipeline on Databricks for continuous automated processing, keeping GA4 data current without manual exports.Databricks · job scheduling
