01 — The Business Driver

The app's analytics couldn't be trusted

A legacy tracking implementation was inflating Vodafone's app penetration metrics by 30-40%, undermining every downstream product decision.

ACCURACY

App penetration was overstated 30-40%

The legacy implementation systematically inflated user counts, distorting the picture of true app adoption.

Faulty metrics = product decisions based on wrong numbers
VISIBILITY

No performance metrics

Without reliable performance metrics, the product team had no baseline to measure improvements against.

No performance metrics = no way to measure improvement
PERSONALIZATION

No A/B testing or personalization

The app had no product analytics layer capable of supporting A/B testing or personalized experiences.

No personalization layer = every user sees the same app
02 — The Obstacles

What had to happen before any test could run

Before any test could run, the new tracking had to be specified, built into the app, and shipped — all inside a two-month window.

SCHEMA

No agreed data contracts

Without a shared data schema, Adobe Analytics couldn't be wired into the app's event structure.

No agreed schema = Adobe Analytics had nothing to wire into
DELIVERY

Remote team, on-site delivery needed

Coordinating agile delivery with a 4-developer app team required on-site presence in Egypt to keep pace.

Remote-only delivery = too slow for a 2-month deadline
TIMELINE

Two-month go-live window

The gap analysis, schema design, and implementation all had to land inside a 2-month deadline.

Audit to go-live = one fixed runway, no slack
